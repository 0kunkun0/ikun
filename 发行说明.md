# ikun库7.1.2 Release Candidate发行说明
**目标平台Windows, Linux和macOS**
**支持架构: x86(Windows, Linux), x86_64(Windows, Linux, macOS), ARM64(Windows, Linux), ARM64EC(Windows, macOS), ARM64E(macOS)**
**[本库GitHub仓库](https://github.com/0kunkun0/ikun)**
**此版本GitHub分支: preview, 可通过`ikun -ip 库名`来安装库**
- 仅供个人, 非营利性组织, 开源项目以及竞赛使用
- 根据GPL 3.0许可证规定, **禁止使用本库进行闭源用途**
---
## 概述
- **ikun库是一个跨平台的C++编程工具库, 支持Windows, Linux和macOS系统, 遵循GPL 3.0许可证**, 此版本是**7.1.2 RC版**, 预计2026年2月发布正式版
 - 提供了丰富的功能, 包括但不限于:
 *文件操作*(files.hpp)
 *数学运算*(maths.hpp)
 *随机数生成*(random.hpp)
 *时间处理*(times.hpp)
 *错误处理*(ikun_stderr.hpp)
 *控制台颜色输出*(console_color.hpp)
 *输入输出整合*(io.hpp)
 *高精度整数运算*(high_precision_digit.hpp)
 *C++标准库整合*(stdc++lib.hpp)等
 - 允许在开源的情况下向ikun库添加扩展功能, 但**必须遵循GPL 3.0许可证规定**

- 开发工具方面, 对开发者提供错误分析器(仅限本库的代码)`ikun_error_analyzer.cpp`和库/项目管理工具`ikun_core.cpp`(编译后应为`ikun(.exe)`)

- 法律方面, ikun库遵循GPL 3.0许可证, **禁止使用本库进行商业闭源用途**

- 备注: 由于本库使用极其新的C++标准(**C++23**), 所以**请保证你的编译器支持C++23**, 否则你就需要修改超级多的代码(本库不存在任何`std::cout`和`std::endl`的代码, 所以修改兼容版本的人有福了)并牺牲`std::print(ln)`的高性能
---
## 新增功能
- 修改`maths.hpp`模块的报错, 使其使用`ikun_stderr.hpp`中的错误抛出机制
 - 全平台

- `random.hpp`修改为使用`random_::Random`类而非一大堆函数
 - 全平台
- `random.hpp`新增`random_double()`函数, 用于处理浮点数随机数
 - 全平台
- `random.hpp`新增`random_bool()`函数, 用于处理布尔随机数
 - 全平台
- `random.hpp`修改错误抛出逻辑, 使用`ikun_stderr.hpp`中的错误抛出机制
 - 全平台

- 为`high_precision_digit.hpp`添加类型转换`to_long_long()`, `to_int()`和与整数/字符串的比较
 - 全平台
- 为`high_precision_digit.hpp`添加基于`ikun_stderr.hpp`的错误处理逻辑
 - 全平台
- 为`ikun_error_analyzer.cpp`添加处理类型为`ikun_maths`的错误, 更新`ikun_hpd`类型错误的错误代码
 - 全平台

## 当前已知的bug
- (**WSL默认不带浏览器, 而`ikun -l`依赖浏览器打开本库GitHub仓库**)在Linux上使用`ikun_core.cpp`时, `ikun -l`无法运行
 - Windows Subsystem for Linux x86_64

- 在Linux上使用`build_lib.py`时, 使用Clang作为编译器时提示`fatal error: 'print' file not found`, **作者认为是环境没配好**, 当前版本的`build_lib.py`中编译命令指定的是C++26标准(print是C++23标准的库), 而且GCC可以正常编译, 所以应该就是作者的环境问题
 - Windows Subsystem for Linux x86_64
 - **如果你的设备上可以用Clang编译成功, 请忽略本条报告**

- (**作者的问题**, 因为作者没有Mac电脑)作者无法测试macOS上的任何功能, 作者正在尝试使用黑苹果虚拟机, 将在7.1.3版本发布前修复(不过由于macOS兼容POSIX API, 所以作者认为这个问题不大), 如果**你使用macOS并发现bug, 请联系作者**
 - macOS x86_64 (Intel & Hackintosh)
 - macOS ARM64E (Apple Silicon)
